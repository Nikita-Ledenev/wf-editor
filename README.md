# Схема рабочего процесса (Workflow Editor)

Редактор рабочего процесса: **таблица состояний** и **диаграмма** связанных стрелками блоков.
Собран как микрофронтенды через **single-spa**:

| Часть                                | Стек                                                    |
| ------------------------------------ | ------------------------------------------------------- |
| Таблица (`packages/table-vue`)       | TypeScript + **Vue 3** + Pinia + SCSS-модули            |
| Диаграмма (`packages/diagram-react`) | TypeScript + **React 19** + Redux Toolkit + SCSS-модули |
| Оболочка (`packages/root-config`)    | single-spa + Vite + Cypress (e2e)                       |
| Общий код (`packages/shared`)        | типы, событийная шина, API-клиент                       |
| Бэкенд (`backend`)                   | Express (отдаёт `data/wf1.json`)                        |

Таблица — владелец данных (читает/пишет на бэкенд). Диаграмма получает состояние и
шлёт события через общую **шину `CustomEvent`** (`packages/shared/src/bus.ts`):
`wf:state` (полное состояние), `wf:select` (выбор шага), `wf:move` (перетаскивание блока).

## Требования

- **Node ≥ 20** (проверено на 22.13). Бэкенд исходно собирался на Node 24.
- Порты: бэкенд `4000`, таблица `4101`, диаграмма `4102`, оболочка `4100`.

## Запуск

```bash
# 1. Зависимости фронта (workspaces)
npm install

# 2. Зависимости и запуск бэкенда (порт 4000)
npm --prefix backend install
PORT=4000 npm --prefix backend run dev

# 3. Всё разом (бэкенд + 3 микрофронта) одной командой
npm run dev
```

Открыть приложение: **http://localhost:4100**
Swagger бэкенда: **http://localhost:4000/swagger**

> Бэкенд слушает IPv4; фронт обращается к `http://127.0.0.1:4000`, чтобы избежать
> проблем с резолвом `localhost` в IPv6 в некоторых браузерах.

## Продакшн-сборка

```bash
npm run build       # собирает оба микрофронта и оболочку (dist/ в каждом пакете)

# предпросмотр собранного (каждый микрофронт со своего origin):
npm --workspace @wf/table-vue   run preview   # :4101
npm --workspace @wf/diagram-react run preview  # :4102
npm --workspace @wf/root-config run preview    # :4100
```

В сборке микрофронты отдаются со своих origin (`base` = их URL), а оболочка
монтирует их по build-карте `src/importMap.json`. CSS-бандлы монтируются/
демонтируются через `cssLifecycleFactory` из `vite-plugin-single-spa`.

## Тесты

```bash
npm run test:unit   # Vitest: стор таблицы + геометрия/редьюсер диаграммы
npm run e2e         # Cypress e2e против запущенного приложения (dev или preview)
```

E2E проходит и против dev-серверов, и против продакшн-сборки (preview).

## Качество кода

```bash
npm run lint        # ESLint (TS + Vue + React)
npm run stylelint   # Stylelint (SCSS + <style> в .vue)
npm run format      # Prettier --write
npm run format:check
```

Pre-commit хук (husky + lint-staged) автоматически прогоняет ESLint/Stylelint/Prettier
по застейдженным файлам. Активируется после `git init` и `npm install`.

## Возможности

- Таблица: sticky-шапка, цвет/координаты/переходы каждого состояния.
- Двусторонняя подсветка: строка ↔ блок.
- Создание состояния (имя «Шаг N» с контролем уникальности, координаты 0,0).
  Цвет выбирается палитрой в шапке (по умолчанию белый); сохраняется через `createStep`.
  Цвет существующего шага не меняется — бэкенд не предоставляет такого эндпоинта.
- Переименование инлайн (двойной клик).
- Удаление с окном отмены: тост «Отменить» (на сервер уходит только по истечении окна).
- При выделении состояния на диаграмме подсвечиваются его связи (соседние блоки и
  инцидентные стрелки), остальное приглушается.
- Диаграмма: блоки по координатам, цветные рамки, стрелки переходов.
- Перетаскивание блоков (координаты сохраняются на сервере).
- Зум колесом к точке курсора и кнопками, авто-вписывание.
- Сортировка по столбцам с сохранением в localStorage.
- Клиентский поиск (фильтрация) по названию.
- Клавиатура в таблице: ↑/↓ — навигация, Enter — создать, Delete — удалить,
  F2 — переименовать, Esc — снять выделение; видимое кольцо фокуса.
